# 测试计划 (TEST_PLAN.md)

以下为 QuickPeek 项目的测试用例清单。

| 用例 ID | 测试模块 | 测试步骤与预期结果 | 是否涉及回归测试 | 状态 |
| :--- | :--- | :--- | :---: | :--- |
| TC-001 | 全局热键 | **步骤**：运行应用，在 Finder 之外的任何地方按下 `Cmd + Shift + Space`。<br>**预期**：成功触发预览流程（若没有选中文件，显示相应提示）。 | 是 | 待测试 |
| TC-002 | Finder 文件检测 | **步骤**：在 Finder 中不选择任何文件，按下触发快捷键。<br>**预期**：浮动窗口显示“未选择任何文件”或通过 Toast 提示。 | 是 | 待测试 |
| TC-003 | 文本文件预览 | **步骤**：在 Finder 中选中一个普通的 `.txt` 文本文件，按下触发快捷键。<br>**预期**：浮动窗口打开并以纯文本样式正确显示其内容。 | 是 | 待测试 |
| TC-004 | Markdown 渲染 | **步骤**：在 Finder 中选中一个包含标题、列表、代码块的 `.md` 文件，按下触发快捷键。<br>**预期**：浮动窗口打开，内容通过 HTML CSS 样式渲染，格式美观，支持滚动。 | 是 | 待测试 |
| TC-005 | 代码高亮展示 | **步骤**：在 Finder 中选中一个 `.swift` 或 `.py` 文件，按下触发快捷键。<br>**预期**：浮动窗口打开，代码显示对应的语法高亮。 | 是 | 待测试 |
| TC-006 | 编辑模式切换 | **步骤**：在预览窗口中，点击编辑按钮或按下快捷键切换到编辑状态。<br>**预期**：文本变为可编辑，侧边显示行号。 | 是 | 待测试 |
| TC-007 | 编辑保存 | **步骤**：在编辑模式下修改文件内容，点击保存（或按下 `Cmd + S`）。<br>**预期**：修改内容成功写入磁盘，重新读取时内容已更新。 | 是 | 待测试 |
| TC-008 | 设置项变更 | **步骤**：在菜单栏图标中打开设置，修改字体大小，点击保存。<br>**预期**：再次预览文件时，字体大小按设置的参数渲染。 | 是 | 待测试 |
| TC-009 | 辅助功能权限引导 | **步骤**：在没有授予辅助功能权限的情况下启动应用。<br>**预期**：弹出提示框引导用户打开系统偏好设置以授予权限。 | 是 | 待测试 |
| TC-010 | 窗口无边框、Toggle关闭、图层弹簧动画与后台回归测试 | **步骤**：1. 运行应用，确认无 Dock 与系统菜单栏图标；2. 选中 Finder 文件，按快捷键呼出，确认窗口以带内容的丝滑弹簧动画**从该文件图标位置**放大弹出（确认无任何先放大后内容闪现的割裂感，也无两段式隐藏闪现），后面无遮罩且无红绿灯；3. 拖拽预览窗口的任意背景空白处，确认可自由移动窗口；4. 在窗口聚焦状态下直接再次按快捷键，确认窗口以收缩动画**飞回该文件图标原位**；5. 多次连续切换。<br>**预期**：1. 后台 Agent 运行成功；2. 窗口为纯粹无边框设计，支持背景空白拖拽，无红绿灯及细线边框残留；3. 聚焦时按热键可立刻 Toggle 关闭，动画能够**精准与内容渲染合并同步**，首帧渲染在动画前完成，且动画结束后 Layer 完美驻留，无任何白屏、先隐藏后突兀闪现的两段式 Bug，极具物理连续性弹簧质感。 | 是 | 待测试 |
| TC-011 | 诊断与提示 Toast 验证 | **步骤**：1. 在系统浅色模式或深色模式下，双击 Option 键触发文件预览；2. 观察屏幕上方是否弹出 Toast 提示（包含文件选中坐标），文字与图标是否高反差清晰可见；3. 在 Toast 消失前，连续多次触发 Toast，观察是否只有一个 Toast 在平滑切换且无重叠堆积。 | 是 | 待测试 |
| TC-012 | Finder 文件物理坐标获取验证 | **步骤**：1. 在 Finder 中选中不同位置的文件项目，双击 Option 键触发预览，读取 Toast 中的坐标；2. 改变 Finder 的视图模式（Icon、List、Column、Gallery）重复测试；3. 打开多个 Finder 窗口（其中一些退到后台，保持各自有选中项），在前台活动窗口中切换选中不同的文件，按下快捷键观察坐标是否实时更新为前台文件的坐标。<br>**预期**：Toast 中的 Position 物理坐标能实时、准确地反馈当前前台活动窗口中选中项目的实际屏幕坐标，绝对不会一律显示固定的 (934, 475) 中心坐标，且诊断源提示应显示“Focused”或“Window 遍历成功”（且不再受后台窗口残留 of 旧选中项干扰导致坐标卡住不更新）。 | 是 | 待测试 |
| TC-013 | Finder 路径检测兼容性与熔断压降验证 | **步骤**：1. 在 Finder 中，选中普通代码小文件，按下快捷键进行预览；2. 进入重命名状态，或在空白处、侧边栏上按下快捷键；3. 连续、快速双击 Option 键十次以上，观察系统是否仍然响应并刷新坐标。<br>**预期**：1. 小文件路径获取 100% 成功，不再弹出“未检测到选中文件”错误；在空白处或侧栏会以当前目录兜底；2. 连续快速触发下，系统绝不发生卡顿，且坐标依然能够实时、高精度地显示，表明 Role 剪枝有效减轻了 95% 以上的无用 IPC 通信，完美杜绝了 macOS `accessibilityd` 安全熔断。 | 是 | 待测试 |
